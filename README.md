#DAY 2:

Today, our instructor introduced us to **Bash (Bourne Again Shell)** and the **Linux Command Line Essentials**, which are fundamental for working in the Linux operating system. We learned that the shell acts as an interface between the user and the Linux kernel by accepting user commands, processing them, and communicating with the kernel to produce the desired output. The instructor also explained that Bash is the default shell in most Linux distributions because it supports scripting, automation, and system administration.

The session also covered the relationship between **hardware, the kernel, the shell, and user applications**. We learned that the hardware performs physical operations, the kernel manages system resources, and the shell allows users to interact with the operating system through commands. This helped us understand how Linux processes user requests.

Another important topic was the **Linux File System Structure**. We learned that Linux follows a hierarchical structure beginning with the **root directory (/)**. The instructor explained the purpose of important directories such as **/home**, **/bin**, **/etc**, **/usr**, **/var**, and **/tmp**, helping us understand how files and system resources are organized.

The practical part of the class focused on the **Linux command line**. We practiced several basic commands, including **pwd**, **ls**, **cd**, **mkdir**, **touch**, **cp**, **mv**, **rm**, **cat**, **clear**, **echo**, **whoami**, **date**, and **history**. These commands are essential for navigating directories, managing files, and working efficiently in the Linux terminal.

Finally, the instructor explained **file and directory permissions** in Linux. We learned about the three permission types—**Read (r)**, **Write (w)**, and **Execute (x)**—and how they are assigned to the **owner**, **group**, and **others** to control access and improve system security.

Overall, today's session provided a solid understanding of Bash, the Linux command line, the file system hierarchy, and file permission management. These concepts form the foundation for advanced topics such as shell scripting, system administration, DevOps, cloud computing, and cybersecurity.

#DAY 3:

Today, our instructor taught us about **redirection** and **pipes**, two important features of the Linux command line. We learned that **redirection** is used to save the output of a command to a file or take input from a file, while **pipes (`|`)** allow the output of one command to be used as the input for another command. These features help users perform tasks more efficiently by combining multiple commands.

The instructor then introduced us to **Bash scripting** and explained how scripts can be used to automate repetitive tasks. We learned the basic structure of a Bash script and practiced writing simple programs, such as displaying our **name, city, and age**, and creating the **multiplication table of a given number**. These examples helped us understand how shell scripts work and how they simplify command-line operations.

The session also covered **file compression** and its importance in reducing file size, saving storage space, and making file transfers easier. We were introduced to common Linux compression and archiving tools used to compress and extract files.

Finally, we learned about **wildcards, escaping characters, quoting, and text processing tools**. The instructor explained how wildcards help in searching and managing multiple files, while escaping characters and quoting ensure that special characters are interpreted correctly by the shell. We also explored basic text processing tools used to search, filter, and manipulate text files efficiently from the command line.

Overall, today's session enhanced our understanding of Linux command-line operations, scripting, file management, and text processing. These concepts are essential for improving productivity and form a strong foundation for advanced Linux administration and shell scripting.
